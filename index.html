SARA

Introduction

SARA (Scheduling Algorithm Reverse Approach) is an approach proposed by PKU OSV Team (Peking University Operating System and Virtualization Team) to reverse the scheduling algorithm of OpenStack. It can determine the activated filters and weighers used by a OpenStack instance through analyzing the AMQP packets between OpenStack components. VisualSARA (also referred as OpenSploit in our paper) is a C++ implementation for SARA method. It is a MFC project running under Windows operating system. It is developed by Visual Studio Ultimate 2015 CTP.

SARA is based on the our paper work "SARA: Exploiting Live VM Migration to Reverse the Scheduling Algorithm in Clouds" (will be published soon), it supports OpenStack Juno. SARA's functionalities includes:

  1. Scheduling simulation for 16 kinds of filters and 3 kinds of weighers.
  2. Filter determination.
  3. Weigher determination.
  4. Calculation of the determination precisions.

Environment

  1. Windows operating system (XP above)
  2. Visual Studio Ultimate 2015 CTP

Dependencies

The libraries used by VisualSARA:

  1. Boost 1.57
  2. JSON Spirit 4.08
  3. CLAPACK 3.2.1
  4. Armadillo 4.650.2

Datasets

Datasets used by the paper:

  • sara_filter_type1 - Datasets for Type-I filters
  • sara_filter_type2 - Datasets for Type-II filters
  • sara_weigher - Datasets for weighers

You should change the strSARAFileBase variable in VMTester.cpp to change the default dataset directory.

The naming convention for dataset files is:

  • sara_host_$HOST_NUMBER$.json - the host metadata file
  • sara_request_$REQUEST_NUMBER$_$HOST_NUMBER$.json - the request specification file
  • sara_result_$ENABLED_FILTER_NUMBER$_$REQUEST_NUMBER$_$HOST_NUMBER$.txt - the scheduling and determination result file

Compile

  1. Install VS2015 CTP(Visual Studio Ultimate 2015 CTP) downloaded from Microsoft website.
  2. Download and Configure the libraries VisualSARA used (Boost, JSON Spirit, CLAPACK, Armadillo), default paths for these libraries in VisualSARA are: C:\lib\boost_1_57_0, C:\lib\json_spirit_v4.08, C:\lib\CLAPACK, C:\lib\armadillo-4.650.2, you can modify these paths based on your need.
  3. Compile the solution in VS2015.

Usage

  1. Switch to the needed dataset directory.
  2. Change the m_iRequestNo, m_iHostNo, m_iFilterNo variables in VMTester.cpp to select input files.
  3. Run VisualSARA.
  4. The result file will be generated in the dataset directory.
